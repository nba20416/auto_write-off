# 智能核销系统 — 产品需求文档 (PRD)

> 版本: v1.1 | 日期: 2026-06-07 | 状态: 已确认，待开发

---

## 1. 项目背景

财务人员每天需要将**销售登款表**与**各银行流水表**进行逐笔核对（核销）。当前为手工操作，效率低且容易遗漏。本系统自动完成**1 对 1 精准金额匹配**，无法自动核销的（0 笔或 ≥2 笔）留待人工处理。

**核心原则：宁缺毋滥。有数字必须对得上，绝不允许猜测或模糊匹配。**

---

## 2. 核心流程

```
销售登款表（每一行）
    │
    ├─ 依次查询各银行文件
    │   ├─ 招商银行.xlsx
    │   ├─ [银行B].xlsx
    │   └─ [银行C].xlsx ...
    │
    ├─ 汇总所有银行中「金额相等」的记录数
    │
    ├─ 匹配数 = 1  → ✅ 自动核销：绿色高亮 + "已完成"
    ├─ 匹配数 = 0  → ⚪ 不处理（未到账）
    └─ 匹配数 ≥ 2  → ⚠️ 不处理（一对多，需人工判断）
```

---

## 3. 数据结构

### 3.1 销售登款表

| 列名 | 说明 |
|------|------|
| `客户姓名` | 客户名称 |
| `时间` | 销售时间 |
| `金额` | 销售金额（千分位 + 两位小数，如 `1,234.56`） |

> 数据从第 2 行开始，第 1 行为表头。

### 3.2 银行流水表

| 列名 | 说明 |
|------|------|
| `付款人姓名` | 付款方名称 |
| `付款时间` | 付款时间 |
| `金额` | 付款金额（千分位 + 两位小数，如 `1,234.56`） |

> 数据从第 2 行开始，第 1 行为表头。
> 每个银行一个独立 `.xlsx` 文件。

### 3.3 当前银行列表

| 银行 | 文件名示例 |
|------|-----------|
| 招商银行 | `招商银行.xlsx` |

> **银行和列名均支持后期增删改查，通过配置文件管理。**

## 4. 功能需求

### 4.1 匹配规则

- **唯一匹配维度：金额**（精确到分，四舍五入保留 2 位小数后比较）
- 金额读取时自动处理千分位逗号和货币符号
- 不考虑日期、户名等其他字段

### 4.2 匹配范围

对于销售登款表的**每一行**：
1. 提取 `金额` 列的值
2. 在**所有**银行文件中查找相同金额的记录
3. 跨银行汇总匹配总条数

### 4.3 处理逻辑

| 匹配数 | 动作 | 视觉效果 |
|--------|------|----------|
| **= 1** | 自动核销 | 整行**绿色背景**，`核销状态` 列写入 `已完成` |
| **= 0** | 不处理 | `核销状态` 列留空 |
| **≥ 2** | 不处理 | `核销状态` 列写入 `待人工处理`（仅标注，不改色） |

### 4.4 新增列

在原表最右侧新增两列：

| 新增列名 | 内容 |
|----------|------|
| `核销状态` | `已完成` / `待人工处理` / （空） |
| `匹配来源` | 匹配到的银行名（如 `招商银行`），用于追溯审计 |

> 注：匹配来源记录银行名即可，不记录行号。因为银行表可能随时更新，行号无意义。

### 4.5 输出

生成新文件，命名规则：`{原文件名}_已核销.xlsx`

终端同步打印汇总：

```
===== 核销汇总 =====
总记录数:        150
已完成:          120  (80.0%)
待人工处理:        8  ( 5.3%)
未匹配:           22  (14.7%)
=====================
```

## 5. 配置管理

通过 `config.json` 管理银行和列名，支持增删改查：

```json
{
  "sales_table": {
    "amount_column": "金额",
    "name_column": "客户姓名",
    "date_column": "时间"
  },
  "bank_tables": {
    "amount_column": "金额",
    "name_column": "付款人姓名",
    "date_column": "付款时间"
  },
  "banks": [
    "招商银行"
  ]
}
```

- 新增银行：在 `banks` 数组中加一行，并准备对应的 `{银行名}.xlsx` 文件
- 修改列名：直接改 JSON 中的字段值
- 删除银行：从数组中移除

---

## 6. 运行方式

**命令行脚本**（推荐）：

```bash
# 基本用法
python reconcile.py 销售登款表.xlsx 招商银行.xlsx

# 多个银行
python reconcile.py 销售登款表.xlsx 招商银行.xlsx 工商银行.xlsx 建设银行.xlsx
```

- 第一个参数固定为销售登款表
- 后续参数为银行流水表（可传入任意数量）
- 银行文件名（去掉 `.xlsx` 后缀）即为银行名

---

## 7. 非功能需求

### 7.1 准确性
- 金额比较统一用 `Decimal` 或 `round(x, 2)`，避免浮点精度问题
- 读取时自动清理千分位逗号、`¥` 符号、首尾空格

### 7.2 安全性
- **不修改原始文件**，结果输出到新文件
- 原始文件只读打开

### 7.3 性能
- 支持单表 10000 行以内流畅处理
- 银行文件数 ≤ 10 个
